# dash-particles v0.0.2

`v0.0.2` turns `dash-particles` into a much more Python-first package.
The headline change is a structured config API that lets Dash users build
tsParticles options with composable Python objects instead of one large nested
dictionary.

## Highlights

- Added a structured config model built around `dp.Options(...)`,
  `dp.Particles(...)`, `dp.Interactivity(...)`, and related helper classes.
- Added component-level convenience arguments including `config=`,
  `background=`, `particles=`, `interactivity=`, `fps_limit=`, and
  `detect_retina=`. These serialize into the underlying tsParticles `options`
  payload for you.
- Added curated `dp.presets.*` helpers for faster setup, including both
  general presets like `stars()` and `connect()` plus richer sample-inspired
  presets such as `among_us()`, `parallax()`, `fontawesome()`,
  `blurred_particles()`, and `hypno_squares()`.
- Expanded the package documentation with getting-started, config-model,
  recipes, migration, and troubleshooting guides.
- Kept raw dictionary configs backward-compatible so existing apps can upgrade
  without rewriting all particle options immediately.
- Upgraded to v3.9.1 of tsParticles to enable more enhancements and features added into the future

## Why This Release Matters

Before `v0.0.2`, the main authoring path was passing a raw tsParticles-shaped
dictionary through `options={...}`. That still works, but the recommended path
is now:

```python
import dash_particles as dp

particles = dp.DashParticles(
    id="particles",
    config=dp.Options(
        background=dp.Background(color=dp.Color("#0f172a")),
        particles=dp.Particles(
            color=dp.Color("#38bdf8"),
            number=dp.ParticleNumber(value=80),
            links=dp.Links(enable=True, color="#38bdf8", opacity=0.3),
            move=dp.Move(enable=True, speed=2),
        ),
    ),
)
```

That change makes configs easier to read, easier to refactor in Python, and
easier to document for Dash users who do not want to live in raw JSON-shaped
objects.

## New API Surface

The main additions in this release are:

- `dp.Options(...)` and `dp.ParticlesOptions(...)` for top-level config objects
- Structured helpers such as `dp.Background`, `dp.Color`,
  `dp.Particles`, `dp.ParticleNumber`, `dp.Links`, `dp.Move`,
  `dp.Interactivity`, `dp.Events`, `dp.Action`, `dp.Size`, `dp.Opacity`,
  `dp.Shape`, `dp.OutModes`, and more
- `dp.presets.*` for reusable starter configurations
- `extra={...}` escape hatches on structured config objects for supported
  tsParticles keys that do not yet have a dedicated Python helper

## Compatibility Notes

- `config=` accepts structured config objects and raw dictionaries
- `options=` is still supported for legacy usage
- `options=` and `config=` cannot be used together in the same component call
- Explicit component overrides like `particles=...` or `background=...` take
  precedence over overlapping keys from `config=`
- `extra={...}` helps with Python helper coverage, but it does not load missing
  frontend plugins on its own

## Runtime Support Notes

`dash-particles` now ships with the `tsparticles` full frontend bundle plus the
image-shape, text-shape, and canvas-mask plugin support used by the documented
examples and presets in this release.

Common background, link, hover, click, image, themeable blur, canvas-mask, and
character-style scenes are covered by the shipped runtime. Some advanced
features outside the documented surface may still require extra frontend work,
and polygon-mask scenes still need the separate polygon-mask plugin.

## Upgrade Guidance

If you already use `dash-particles`, the easiest upgrade path is:

1. Keep your existing `options={...}` config working as-is.
2. Move new work to `config=dp.Options(...)`.
3. Convert only the sections you touch most often first.
4. Use `extra={...}` while bridging unsupported keys.

If your current setup resembles a common visual pattern, you may be able to
replace much of it with a preset and a small override.
