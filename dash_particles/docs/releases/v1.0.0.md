# dash-particles v1.0.0

`v1.0.0` is the stable Python package release for `dash-particles`. It keeps the
Dash component simple to render while making the recommended authoring path
Python-first and documented end to end.

## Highlights

- Stabilized the structured config API built around `dp.Options(...)`,
  `dp.Particles(...)`, `dp.Interactivity(...)`, and related helper classes.
- Kept curated `dp.presets.*` helpers as the fastest path to a working
  background, with overrides available through the same structured API.
- Added runtime auto-loading so simple scenes use smaller packaged tsParticles
  tiers, while richer scenes can load the packaged full tier on demand.
- Preserved raw dict compatibility through `config={...}` and legacy
  `options={...}` for existing apps.
- Refreshed the docs around installation, migration, recipes, troubleshooting,
  and runtime support boundaries.

## Upgrade Notes

Existing apps using raw tsParticles dictionaries can continue to run without a
full rewrite. New work should prefer `config=dp.Options(...)`, and apps that need
unsupported tsParticles keys can keep using `extra={...}` on structured objects
or pass raw dict sections directly.

Use `runtime="full"` only when you want to force the broadest packaged runtime.
For most apps, the default `runtime="auto"` should pick the smallest packaged
tier that matches the supplied config.

## Suggested Release Summary

`dash-particles v1.0.0` stabilizes the Python-first config API, ships curated
presets, keeps legacy dict configs compatible, and documents packaged
tsParticles runtime auto-loading for Dash apps.
